# Что это за программа

Маленький помощник, который собирает черновик отчёта *ehitise auditi aruanne* в Word — по тем же 14 разделам, по которым Вы делаете их сегодня вручную.

Программа **не подписывает** отчёт за Вас и **не отправляет** его в *ehitisregister*. Подпись и загрузка — по-прежнему Ваши действия.

# Как запустить

Один раз на ноутбуке:

```bash
cd ~/projects/tadf
uv sync
```

Каждый раз когда хотите начать работу:

```bash
cd ~/projects/tadf
uv run streamlit run app/main.py
```

Откроется браузер. Если не открылся — зайдите вручную на <http://localhost:8501>.

# Пять шагов одного аудита

## 1. Новый аудит

- Введите номер по порядку и год (например, 5 / 2026).
- Выберите тип (`EA`, `EP`, `TJ`, `TP`, `AU`) и подтип (`kasutuseelne` / `korraline` / `erakorraline`).
- Дату осмотра.
- Кто составлял (Auditi koostas) и кто проверяет / отвечает (Auditi kontrollis — это Вы по умолчанию, kutsetunnistus 148515).
- Тельджа (заказчик) — название, контакты.

Если оставите «Цель аудита» пустой — программа подставит стандартный текст по подтипу.

## 2. Здание

- Адрес, kataster, EHR-код.
- Год постройки, реконструкция.
- Размеры: ehitisealune pind, высота, объём, этажность.
- Tulepüsivusklass (TP-1 / TP-2 / TP-3) — если применимо.
- Если здание построено до 2003 — поставьте галочку «Pre-2003» и заполните примечание.

## 3. Находки

Это сердце аудита. Добавляйте наблюдения по разделам:
- Раздел 4 — общая ülevaatus
- Раздел 5/6 — архитектура и конструкции (вундамент, стены, перекрытия, крыша…)
- Раздел 7 — техносистемы (вода, электричество, отопление, вентиляция)
- Раздел 8 — пожарная безопасность
- **Раздел 11 — Kokkuvõte (только Вы пишете)**
- **Раздел 14 — Lõpphinnang (только Вы пишете)**

Без хотя бы одной записи в разделах 11 и 14 программа **не даст** собрать отчёт — это требование §5.

Для каждой находки укажите серьёзность (📝 инфо / ⚠️ мелкое / ❗ существенное / 🛑 опасность) и при желании — рекомендацию и ссылки на закон.

## 4. Фото

Загрузите фотографии. Каждое фото получит подпись (по умолчанию — имя файла, лучше переписать) и номер раздела (4, 5, 6, …, 16 — общий блок «Fotod»).

## 5. Готовый отчёт

- Программа проверит, заполнены ли все обязательные по закону поля (§5).
- Нажмите «💾 Сохранить аудит в БД» — данные попадут в локальную базу для повторного открытия позже.
- Нажмите «📄 Собрать отчёт (.docx)» — будет создан файл `data/audits/<id>/draft.docx`.
- Скачайте файл по кнопке «⬇️ Скачать draft.docx».

# Что делать с готовым черновиком

1. Откройте `draft.docx` в Word или LibreOffice.
2. Поправьте формулировки, добавьте таблицы или фото в нужном месте, если это нужно.
3. Сохраните как PDF (`File → Export to PDF` в LibreOffice, или `File → Save As → PDF` в Word).
4. Подпишите PDF через DigiDoc — получится `.asice` файл.
5. Откройте <https://ehitisregister.ee> и загрузите `.asice`.

# Где хранятся файлы

Каждый аудит лежит в своей папке `data/audits/<номер>/`:
- `draft.docx` — собранный черновик
- `context.json` — все исходные данные (для повторной сборки и для 7-летнего архива)
- `photos/` — оригиналы фотографий

База данных — файл `tadf.db` в корне проекта. Можно скопировать его на флешку для бекапа.

# Если что-то не так

- **Программа жалуется, что поле не заполнено** — она подсказывает, на какой странице это поле находится. Вернитесь и заполните.
- **Не получается открыть фото** — проверьте, что формат `.jpg`, `.png` или `.webp`.
- **Программа не запускается** — позвоните Антону.

# Что появится в следующих версиях

(Не надо ничего делать — просто чтобы знать.)

- Подсказки от ИИ для длинных разделов (раздел 6, 7, 8) — Вы пишете тезисы, ИИ предлагает текст, Вы принимаете или отклоняете.
- Подпись `.asice` прямо из программы (Smart-ID / ID-card).
- Доступ из браузера с любого устройства (когда захотите — программа переедет на сервер в Финляндии).
- Прямая отправка в *ehitisregister* через X-tee — если получим разрешение от RIA.
